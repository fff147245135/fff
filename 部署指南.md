# GitHub Pages 部署指南

## 步骤一：在 GitHub 上创建仓库

1. 访问 [GitHub](https://github.com) 并登录你的账号
2. 点击右上角的 **"+"** 按钮，选择 **"New repository"**（新建仓库）
3. 填写仓库信息：
   - **Repository name**（仓库名）：例如 `graduation-portfolio` 或 `毕业创作`
   - **Description**（描述）：可选，例如 "毕业创作作品集网站"
   - 选择 **Public**（公开，GitHub Pages 免费版需要公开仓库）
   - **不要**勾选 "Initialize this repository with a README"（我们已经有了文件）
4. 点击 **"Create repository"**（创建仓库）

## 步骤二：上传文件到 GitHub

### 方法一：使用 GitHub Desktop（最简单，推荐）

1. 下载并安装 [GitHub Desktop](https://desktop.github.com/)
2. 打开 GitHub Desktop，登录你的 GitHub 账号
3. 点击 **"File"** → **"Add Local Repository"**
4. 点击 **"Choose..."**，选择你的项目文件夹（`F:\设计作品\摄影\毕业创作`）
5. 如果提示需要初始化仓库，点击 **"create a repository"**
6. 在左侧会看到所有文件，在底部填写提交信息，例如："初始提交：毕业创作作品集网站"
7. 点击 **"Commit to main"**（提交到主分支）
8. 点击右上角的 **"Publish repository"**（发布仓库）
9. 选择你刚才在 GitHub 上创建的仓库，点击 **"Publish repository"**

### 方法二：使用 Git 命令行

如果你已经安装了 Git，在项目文件夹中打开终端/命令行，执行以下命令：

```bash
# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交文件
git commit -m "初始提交：毕业创作作品集网站"

# 添加远程仓库（将 YOUR_USERNAME 和 YOUR_REPO_NAME 替换为你的实际信息）
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

### 方法三：直接在 GitHub 网页上传（适合少量文件）

1. 在 GitHub 仓库页面，点击 **"uploading an existing file"**
2. 将项目文件夹中的所有文件拖拽到页面
3. 在底部填写提交信息
4. 点击 **"Commit changes"**

## 步骤三：启用 GitHub Pages

1. 在 GitHub 仓库页面，点击上方的 **"Settings"**（设置）标签
2. 在左侧菜单中找到 **"Pages"**（页面）选项
3. 在 **"Source"**（源）部分：
   - 选择 **"Deploy from a branch"**
   - Branch（分支）选择 **"main"**（或 "master"）
   - Folder（文件夹）选择 **"/ (root)"**
4. 点击 **"Save"**（保存）
5. 等待几分钟，GitHub 会显示你的网站地址，格式为：
   ```
   https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/
   ```

## 步骤四：访问你的网站

1. 等待 1-2 分钟让 GitHub 完成部署
2. 访问 GitHub 显示的网址
3. 如果显示 404，等待几分钟后刷新页面

## 更新网站内容

以后如果需要更新网站内容：

### 使用 GitHub Desktop：
1. 修改文件后，在 GitHub Desktop 中会看到更改
2. 填写提交信息
3. 点击 **"Commit to main"**
4. 点击 **"Push origin"** 推送到 GitHub
5. 几分钟后网站会自动更新

### 使用 Git 命令行：
```bash
git add .
git commit -m "更新内容描述"
git push
```

## 常见问题

### Q: 网站显示 404 错误？
A: 
- 等待 5-10 分钟后再试（首次部署需要时间）
- 检查 Settings → Pages 中的配置是否正确
- 确保仓库是 Public（公开）的

### Q: 图片显示不出来？
A: 
- 确保所有图片文件都已上传到 GitHub
- 检查图片路径是否正确（区分大小写）
- 清除浏览器缓存后刷新

### Q: 如何自定义域名？
A: 
- 在仓库 Settings → Pages 中可以设置自定义域名
- 需要配置 DNS 记录

### Q: 网站更新后没有变化？
A: 
- 清除浏览器缓存（Ctrl+F5 强制刷新）
- 等待几分钟让 GitHub 完成部署

## 提示

- 📝 每次更新后，GitHub 通常需要 1-2 分钟来重新部署
- 🔗 你的网站地址是永久的，可以分享给任何人
- 💡 建议定期备份你的文件到本地
